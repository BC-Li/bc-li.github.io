Openpose 配置小记

发表于 2020-03-28 更新于 2020-05-03 阅读次数：

记录了一点 Openpose 的配置过程及踩过的坑

这些文字已经是编辑的第二遍了，因为上个文件因为脑袋太晕在重启的时候没保存…
Openpose 是我们在新生研讨课中选择的课题。
大概这是我用时最长，产出最少的一个项目了，光是安装就一直忙到现在，API的调用遥遥无期。
其实之前有跑通过的，但是由于系统更新搞掉了驱动，所以一直没有再次运行成功。
Ubuntu对于NVIDIA显卡驱动的支持程度差得超乎我的想象，所以下次掉驱动我可能不会像现在这样挣扎了好几周才选择重装Ubuntu，而是直接插上我的启动盘（可以，不和他多BB—某主播）

这是具体的安装教程，照着走就可以，十分详细，不过里面的一些坑还是很闹心的。

  • CMake 版本不要按这个来，否则会出现报错：NOTFOUNDCUDA_cublas_device_LIBRARY。
    解决方案
    如果你安装了旧版的 CMake，那就直接删掉旧的重装一下就可以了，软连接之后 CMake - gui就能进入 gui 界面，不要安装 CMake-QT-gui ，那个是旧版的。

  • CUDA9 最多支持到 Ubuntu 16 ，没有 Ubuntu 1804 的 deb 包 / Run File
    解决方案：实测 CUDA 10 也可以正常跑，不过，跑别的项目可能会崩，照我来讲，我可能会选择再次重装系统。 （我的配置是 i7 - 9750H + GTX 1650 )

  • 如果你选择了 BUILD-PYTHON, 不事先 clone 一个 Pybind 肯定出错。因为 git clone 下来的仓库： 3rdparty 文件夹是空的。

  • 不要用 Ubuntu 的自动更新
    别问我是怎么知道的。

  • 更多的稀奇古怪的 BUG，建议你学会看错误信息而不是复制粘贴一堆没用的 log 在百度上，能搜到才见鬼。

  • 复制粘贴类似环境变量 EXPORT 的时候，看准了里面要填写你自己的路径，复制别人的路径没用…

  • USE Google instead of Baidu / CSDN
    你懂的。

其实我是还能写更多的，但是现在太忙了，下周还会有代数结构的小测，需要复习。
这次用 4.5 H 把整个过程又重走了一遍，比刚开始花 10 天只搞一个这种…的东西强多了。
可能带给我的唯一收获是少到可怜的 Linux 基本操作技巧和 好脾气 。
不过有了好脾气很多东西不就很好解决了嘛 2333
哦，还有我打字的速度明显增快了（刚开始都不知道 Shift + Ctrl + C / V 是 Linux 下的复制粘贴，手打了不知道多少命令）。有时间考虑搞个机械键盘（搞不好会被室友打哦…）
